You are a senior full-stack engineer and product designer. Build a complete, 
production-quality web application called "BioIntel" — an AI-powered Species 
Intelligence & Research Platform.

This is a dark, cinematic, futuristic scientific platform. Think National 
Geographic meets Linear.app meets a SpaceX mission interface. Every pixel 
should feel premium and alive.

====================================================
TECH STACK — USE EXACTLY THESE
====================================================

Frontend:
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Framer Motion (animations)
- Recharts (data charts)
- React Force Graph (ecosystem network viz)
- Lucide React (icons)

Backend:
- FastAPI (Python)
- PostgreSQL + pgvector extension
- Redis (via Upstash free tier)
- Celery (async task queue)
- SQLAlchemy ORM

AI / ML:
- Ollama (local LLM inference)
- Llama 3.1 8B model (via Ollama)
- BioCLIP (species identification - ONNX)
- YOLOv8 nano (object detection - first pass)
- BGE-M3 (text embeddings - ONNX)
- Sentence Transformers

Package manager: pnpm (frontend), pip (backend)

====================================================
TYPESCRIPT TYPES (lib/types.ts)
====================================================

Define all types:

interface Species {
  id: string
  commonName: string
  scientificName: string
  kingdom: string
  phylum: string
  class: string
  order: string
  family: string
  genus: string
  iucnStatus: 'EX' | 'EW' | 'CR' | 'EN' | 'VU' | 'NT' | 'LC' | 'DD'
  habitat: string
  lifespan: string
  diet: 'Carnivore' | 'Herbivore' | 'Omnivore' | 'Insectivore'
  topSpeed: string
  population: string
  populationTrend: 'Increasing' | 'Stable' | 'Declining' | 'Unknown'
  imageUrl: string
  thumbnailUrl: string
}

interface IntelligenceReport {
  speciesId: string
  dimension: 'evolution' | 'anatomy' | 'behavior' | 'ecosystem' | 'conservation'
  content: string
  citations: Citation[]
  confidence: number
  generatedAt: string
  cached: boolean
}

interface Citation {
  source: string
  url: string
  title: string
  year: number
}

interface IdentificationResult {
  species: Species
  confidence: number
  top5: Array<{ species: Species; confidence: number }>
  processingTime: number
  model: string
}

interface EcosystemNode {
  id: string
  group: 'focal' | 'prey' | 'predator' | 'competitor' | 'scavenger' | 'habitat'
  val: number
}

interface EcosystemLink {
  source: string
  target: string
  type: 'predation' | 'competition' | 'scavenging' | 'habitat'
}

====================================================

   above. Do not use Lorem Ipsum anywhere.

2. ALL COLORS: Use the CSS variables defined above.
   Never hardcode hex colors in components — 
   always reference the variable.

3. DARK MODE: The site is dark-mode-only. Do not 
   implement a light mode toggle.

4. FONTS: Load via next/font/google in layout.tsx 
   for performance. Apply as CSS variables.

5. IMAGES: For mock species images, use 
   picsum.photos with consistent seeds:
   Bengal Tiger: https://picsum.photos/seed/tiger/800/600
   Use real images when API is connected.

6. NO EXTERNAL UI LIBRARIES: Do not use shadcn/ui, 
   Material UI, Chakra, or Radix. Build all 
   components from scratch using Tailwind.

7. PERFORMANCE: 
   - Use next/image for all images
   - Use dynamic imports for heavy components 
     (React Force Graph, Recharts)
   - Add loading skeletons for the intelligence 
     tab content

8. LOADING STATES: 
   When NEXT_PUBLIC_USE_MOCK=false and API calls 
   are made, show skeleton loaders that match 
   the layout of the content they're replacing.

   NEXT_PUBLIC_USE_MOCK=true so the full UI

    2. UI primitives (GlowCard, DataPill, etc.)
    3. Navbar + Footer
    4. Landing page sections
    5. Analyze page
